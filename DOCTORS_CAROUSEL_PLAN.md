## 2. Загрузка фото

- Скачать оригиналы (по возможности без обрезки `-300x350` в URL, т.е. взять полноразмерный
  файл, убрав суффикс `-300x350` из URL, если полноразмерный существует; иначе — как есть).
- Сохранить в `F:\vision\images\doctors\` под именами из таблицы выше.
- Проверить, что все 15 файлов скачались и открываются (не битые).

## 3. Верстка секции-карусели на главной странице (`index.html`)

- Новая секция `<section class="section section--light" id="doctors">` — разместить сразу
  после секции `#top` (hero) и перед секцией `#how-to-read` («Как читать карточку
  результатов»).
- Заголовок секции: «Наши врачи-офтальмологи».
- Горизонтальная карусель (скролл со snap, кнопки-стрелки влево/вправо, поддержка свайпа
  на мобильных — как нативный `overflow-x: auto; scroll-snap-type: x mandatory`, без
  сторонних библиотек, в духе остального проекта на чистом JS).
- Каждая карточка: фото врача, ФИО, специализация под фото. Модальное окно не нужно —
  вся информация, которая есть на dmclinic.uz (ФИО + специализация), умещается прямо
  на карточке.
- Карточки — статическая разметка прямо в `index.html` (15 карточек), без генерации через
  JS-массив — это проще и соответствует объёму данных (только фото/ФИО/специализация, без
  доп. текста, который оправдывал бы вынесение в JS-объект).

## 4. JS (`script.js`)

- Только логика прокрутки карусели: стрелки влево/вправо (`scrollDoctorsCarousel(dir)`),
  скролл через `scrollBy`. Поддержка свайпа/колеса — за счёт нативного
  `overflow-x: auto; scroll-snap-type: x mandatory`, без доп. JS.

## 5. CSS (`styles.css`)

- Стили для `.doctors-carousel`, `.doctor-card`, стрелок навигации — в стилистике уже
  существующих секций (`--light`/`--dark`, отступы, шрифты, скругления, тени — как у соседних
  блоков).
- Адаптив для мобильных (карусель со свайпом, карточки меньшего размера).

## 6. Проверка

- Открыть страницу локально, проверить:
  - Прокрутку карусели (стрелками и свайпом/колесом).
  - Корректность фото и подписей для всех 15 врачей.
  - Адаптивность на мобильной ширине экрана.
